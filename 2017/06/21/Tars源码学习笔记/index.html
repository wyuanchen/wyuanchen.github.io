然后在看setHandle(adapter[i]);

再看adapter->setHandle<ServantHandle>();

再看_pEpollServer->setHandleGroup<T>(_handleGroupName, _iHandleNum, this);

上面的调用的顺序有点像是委托模式是：

  1. Application对象调用成员函数setHandle(adapters[i])把一个Adapter传进去
  2. adapter[i]对象调用自己成员函数setHandle<ServantHandle>()
  3. 最后由_pEpollServer对象来调用自己成员函数setHandleGroup<T>(_handleGroupName, _iHandleNum, this);来完成实际的操作。

AdapterProxy

主要方法为：

  • invoke
  • finishInvoke

并没有负责真正IO的读写，只是调用Transver来执行真正IO

也没有负责真正的数据编解码，而是调用ObjectProxy来实现协议编解码

AdapterProxy的主要作用应该是调用ObjectProxy进行对数据进行协议的编码，然后调用Transver来把编码后的数据输出。

然后负责调用响应数据（是已经经过解码的）自身的回调函数进行i响应，至于AdapterProxy有没有负责响应数据流的解码我还不清楚，不过估计应该是没有的。

invoke(ReqMessage * msg)

finishInvoke(ReqMessage * msg)

finishInvoke有3个版本：

  • void finishInvoke(ResponsePacket &rsp)
  • void finishInvoke(ReqMessage * msg)
  • void finishInvoke(bool bTimeout)

不过主要操作还是放在void finishInvoke(ReqMessage * msg)
